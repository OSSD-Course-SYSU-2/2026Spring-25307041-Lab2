# ArkUI预览功能使用指南

## 概述

本项目已配置完整的ArkUI预览功能，支持页面预览、组件预览、多断点预览和卡片预览四种模式。本文档将详细介绍如何使用这些预览功能。

## 预览功能类型

### 1. 页面预览

页面预览通过`@Entry`装饰器实现，用于预览整个页面的UI效果。

**使用方法**：
1. 在DevEco Studio中打开`MainPage.ets`文件
2. 点击右侧边栏的**Previewer**按钮
3. 预览器将显示整个页面的UI效果

**配置示例**：
```typescript
@Entry
@Component
@Preview({
  title: '主页面 - 默认预览',
  deviceType: 'phone',
  width: 1080,
  height: 2340,
  colorMode: 'light',
  dpi: 480,
  locale: 'zh_CN',
  orientation: 'portrait',
  roundScreen: false
})
struct MainPage {
  // 页面内容
}
```

### 2. 组件预览

组件预览通过`@Preview`装饰器实现，用于独立预览单个组件的UI效果。支持配置多个预览场景。

**使用方法**：
1. 在DevEco Studio中打开组件文件（如`MazeView.ets`）
2. 点击右侧边栏的**Previewer**按钮
3. 预览器将显示该组件的所有预览实例

**配置示例**：
```typescript
@Component
@Preview({
  title: '迷宫视图 - 默认手机',
  deviceType: 'phone',
  width: 360,
  height: 640,
  colorMode: 'light',
  dpi: 480,
  locale: 'zh_CN',
  orientation: 'portrait',
  roundScreen: false
})
@Preview({
  title: '迷宫视图 - 平板',
  deviceType: 'tablet',
  width: 768,
  height: 1024,
  colorMode: 'light',
  dpi: 480,
  locale: 'zh_CN',
  orientation: 'portrait',
  roundScreen: false
})
export struct MazeView {
  // 组件内容
}
```

**注意事项**：
- 单个文件最多可以使用10个`@Preview`装饰器
- 每个`@Preview`装饰器会创建一个独立的预览实例
- 预览实例之间互不干扰，可以同时显示

### 3. 多断点预览

多断点预览可以同时展示8个典型档位断点的预览画面，用于验证响应式布局。

**使用方法**：
1. 确保DevEco Studio版本 ≥ 26.0.0 Beta1
2. 打开配置了`@Entry`的页面文件
3. 启动预览器
4. 在预览器工具栏中点击**多断点预览**按钮
5. 预览器将同时显示8个断点的预览画面

**8个典型断点档位**：
- **xs**: 320x568 (超小屏手机)
- **sm**: 360x640 (小屏手机)
- **md**: 411x731 (中等手机)
- **lg**: 600x1024 (大屏手机/小平板)
- **xl**: 768x1024 (平板竖屏)
- **2xl**: 1024x768 (平板横屏)
- **3xl**: 1280x800 (大平板/小笔记本)
- **4xl**: 1920x1080 (大屏设备)

**注意事项**：
- 多断点预览不支持实时预览和极速预览
- 如果图片太大或太多，预览时可能无法显示
- 代码修改后，所有断点会同步刷新

### 4. 卡片预览

卡片预览用于预览卡片组件的UI效果。

**使用方法**：
1. 创建卡片文件（如`WidgetCard.ets`）
2. 选中卡片文件
3. 点击右侧边栏的**Previewer**按钮
4. 预览器将显示卡片的预览效果

## @Preview装饰器参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| title | string | 'Component' | 预览组件的名称 |
| deviceType | string | 'phone' | 预览设备类型（phone/tablet/default） |
| width | number | 1080 | 预览设备的宽度（单位：px） |
| height | number | 2340 | 预览设备的高度（单位：px） |
| colorMode | string | 'light' | 显示模式（light/dark） |
| dpi | number | 480 | 预览设备的屏幕DPI值 |
| locale | string | 'zh_CN' | 预览设备的语言（如zh_CN、en_US） |
| orientation | string | 'portrait' | 预览设备的横竖屏状态（portrait/landscape） |
| roundScreen | boolean | false | 设备的屏幕形状是否为圆形 |

## 预览器交互功能

### 代码与UI联动

预览器支持代码编辑器、UI界面和组件树三者之间的联动：

1. **选中UI组件**：在预览器UI界面中点击组件，代码编辑器会自动定位到对应代码，组件树也会高亮显示对应节点

2. **选中代码**：在代码编辑器中选中代码块，预览器UI界面会高亮显示对应组件，组件树也会选中对应节点

3. **选中组件树节点**：在组件树中点击节点，代码编辑器会定位到对应代码，UI界面会高亮显示对应组件

### 设备切换

在预览器工具栏中可以切换不同的预览设备：

1. 点击设备选择器
2. 选择预设设备类型（如Phone、Tablet等）
3. 预览界面会自动适配新设备

### 界面缩放

预览器支持界面缩放功能：

- **鼠标滚轮缩放**：使用鼠标滚轮可以缩放预览界面
- **快捷键缩放**：使用快捷键（如Ctrl++、Ctrl+-）可以缩放预览界面
- **缩放重置**：点击缩放重置按钮可以恢复默认缩放比例

## 常见问题与解决方案

### Q1: 预览器无法启动

**可能原因**：
- DevEco Studio版本过低
- 项目配置错误
- 装饰器语法错误

**解决方案**：
1. 检查DevEco Studio版本是否 ≥ 4.0
2. 检查项目配置是否正确
3. 检查装饰器语法是否正确

### Q2: 预览界面显示异常

**可能原因**：
- 组件依赖的参数未正确注入
- 组件使用了不支持预览的API

**解决方案**：
1. 确保组件的所有依赖参数都已正确配置
2. 避免在组件中使用不支持预览的API

### Q3: 多断点预览无法使用

**可能原因**：
- DevEco Studio版本 < 26.0.0 Beta1
- 页面未配置`@Entry`装饰器

**解决方案**：
1. 升级DevEco Studio到最新版本
2. 确保页面已配置`@Entry`装饰器

### Q4: 预览刷新缓慢

**可能原因**：
- 项目代码量过大
- 预览器内存占用过高

**解决方案**：
1. 关闭不必要的预览实例
2. 重启DevEco Studio
3. 清理项目缓存

## 最佳实践

1. **合理使用@Preview装饰器**：单个文件不超过10个`@Preview`装饰器

2. **配置多种预览场景**：为组件配置不同设备、不同方向的预览，确保UI在各种场景下都能正常显示

3. **使用组件片段预览**：对于依赖参数注入的组件，建议定义一个组件片段进行预览

4. **定期验证预览效果**：在开发过程中定期使用预览器验证UI效果，及时发现和修复问题

5. **关注性能指标**：关注预览器的启动时间和刷新时间，确保开发体验流畅
